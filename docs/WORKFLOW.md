## Flow Summary

### Complete End-to-End Workflow

1. **Scraping** â†’ `bd run` â†’ Fetch from 1TamilMV â†’ Save to DB
2. **Check Duplicates** â†’ `bd download --check-duplicates` â†’ Mark lower quality/size as duplicates
3. **Downloading** â†’ `bd download` â†’ qBittorrent â†’ downloads/temp/ (skips duplicates)
4. **Move Completed** â†’ `bd move-completed` â†’ downloads/completed/
5. **Process Video** â†’ `bd process` â†’ mkvmerge (Tamil only) â†’ downloads/processed/
6. **Scan Episodes** â†’ `bd episodes --scan --import-db` â†’ episodes table
7. **AI Matching** â†’ `bd --finder-all` â†’ Comprehensive metadata (IMDB + TMDB) via poster analysis
8. **JojoPlayer** â†’ `bd jojoplayer` â†’ Upload & get streaming links
9. **Status Check** â†’ `bd check-status` â†’ Delete files when completed

### Episode Management Flow

1. **Episode Scan** â†’ Regex extracts info â†’ gpt-5-nano validates unclear cases
2. **Import to DB** â†’ Creates series/seasons/episodes tables
3. **TMDB Matching** â†’ Simple search or AI-powered poster analysis
4. **AI Fallback Chain**: gpt-5-nano (fast) â†’ TMDB API â†’ gpt-5.2 (smart)

### JojoPlayer Integration Flow

1. **Process** â†’ mkvmerge keeps Tamil audio only
2. **Scan** â†’ Import episodes to database with file_path
3. **Upload** â†’ Send to JojoPlayer API â†’ Get streaming URL
4. **Monitor** â†’ Check API status periodically
5. **Cleanup** â†’ Delete files when:
   - Status = "completed" â†’ Delete file + mark deleted=1
   - API = "video not found" â†’ Delete file only (save space)
